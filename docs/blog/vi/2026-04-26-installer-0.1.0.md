# Giới thiệu `vietime-installer` 0.1.0 — gõ tiếng Việt trên Linux trong một cú pháp

*Xuất bản 26 tháng 4, 2026*

Sáu tháng trước chúng tôi ra mắt `vietime-doctor` — công cụ chẩn đoán
chỉ đọc, cho bạn biết **vì sao** gõ tiếng Việt trên Linux bị hỏng.
Doctor rất giỏi xác định nguyên nhân. Cái nó cố tình **không** làm là
tự sửa cho bạn. Đó là thiết kế: Phase 1 phải tạo được niềm tin bằng
cách không bao giờ chạm vào hệ thống.

Hôm nay chúng tôi ra mắt công cụ đồng hành. `vietime-installer` 0.1.0
là mảnh còn lại — lấy khối lệnh sửa lỗi mà Doctor in ra và **chạy nó**
cho bạn, atomic, kèm trail rollback đàng hoàng.

Từ bản Ubuntu mới cài đến gõ được tiếng Việt trong khoảng 2 phút:

```bash
vietime-installer install fcitx5-bamboo
# … một lần hỏi sudo …
# Logout rồi login lại.
```

Xong.

## Vì sao cần công cụ thứ hai?

Công thức setup Fcitx5 + Bamboo trên Ubuntu hiện tại, ngắn nhất có
thể:

1. Gỡ hoặc disable IBus.
2. `sudo apt install fcitx5 fcitx5-bamboo`.
3. Chỉnh `/etc/environment` (hoặc `~/.profile`) với 5 env vars mới.
4. `im-config -n fcitx5`.
5. `systemctl --user enable --now fcitx5`.
6. Logout và login lại.
7. Mở `fcitx5-configtool` và thêm Bamboo.
8. Thử gõ và cầu mong.

Mỗi bước có một kiểu thất bại riêng. Bỏ bước 3 thì GTK không nói
chuyện được với daemon. Bỏ bước 4 thì `im-config` giữ framework cũ.
Bỏ bước 5 thì Fcitx5 không tự chạy. Bỏ bước 6 thì GTK vẫn đọc env cũ
từ phiên trước.

Installer gom tất cả thành một lệnh, ghi mọi thay đổi vào một snapshot
manifest, và cho bạn undo toàn bộ bằng `vietime-installer uninstall`.

## Những gì có trong 0.1.0

- **Bốn combo MVP**: `fcitx5-bamboo`, `fcitx5-unikey`, `ibus-bamboo`,
  `ibus-unikey`.
- **Họ Debian**: Ubuntu 22.04+, Debian 12+, Pop!_OS 22.04+. Fedora và
  Arch sẽ có ở v0.2 (kiến trúc đã hỗ trợ plugin theo distro — v0.1
  chỉ kèm `AptOps`).
- **Dry-run dễ thấy**: `--dry-run` in ra plan mà không thay đổi gì.
- **Rollback atomic**: mỗi lần chạy ghi snapshot ở
  `~/.config/vietime/snapshots/<ts>/` gồm manifest TOML, file backup
  được xác thực bằng SHA-256, và danh sách package installer đã thêm
  (để rollback chỉ gỡ *những cái đó*, không đụng vào gói bạn có sẵn).
- **An toàn với SIGINT**: Ctrl+C giữa các bước để lại manifest cờ
  `incomplete = true`; `rollback --force` dọn dẹp.
- **Không bao giờ cache mật khẩu**: installer dùng credential cache
  riêng của `sudo`. Nó chạy `sudo -v` một lần đầu install và để sudo
  hỏi trên TTY.
- **Tích hợp với Doctor**: `vietime-installer verify` gọi
  `vietime-doctor check` để kiểm tra khói sau khi cài.

## Thiết kế bạn không nhìn thấy

Ruột của installer là ba tầng:

1. **Planner** — hàm thuần `(PreState, Goal) -> Plan`. Nhận ảnh chụp
   hệ thống (tái sử dụng các detector của Doctor) và mục tiêu người
   dùng, trả về danh sách `Step` có cấu trúc. Hoàn toàn xác định,
   roundtrip qua TOML, có golden snapshot test.
2. **Executor** — nhận `Plan` và thực thi. Bất biến quan trọng:
   *ghi trước, biến đổi sau*. Artifact rollback của mỗi step được ghi
   vào manifest **trước** khi mutation chạy. SIGKILL giữa hai bước
   chỉ để lại một rollback target no-op, không bao giờ là thay đổi
   chưa ghi.
3. **Trait `PackageOps`** — abstraction async cho package manager.
   Hôm nay chỉ có `AptOps`. `DnfOps` và `PacmanOps` chỉ cần implement
   trait là xong.

Snapshot được address-theo-nội-dung bằng file sidecar SHA-256, nên
khôi phục backup cũ 10 ngày sẽ phát hiện bit-rot trước khi làm hỏng
`/etc/environment`. Edit env-file được bọc trong marker:

```
# >>> VietIME managed start >>>
GTK_IM_MODULE=fcitx
QT_IM_MODULE=fcitx
XMODIFIERS=@im=fcitx
# <<< VietIME managed end <<<
```

Uninstall chỉ gỡ **những gì nằm giữa marker**. Edit của bạn ở trên
hoặc dưới block được giữ nguyên.

## Tiếp theo là gì?

Phase 2 chứng minh kiến trúc trên một họ distro. Phase 3 mở ngang:

- **INS-50 / INS-51**: `DnfOps` (Fedora) và `PacmanOps` (Arch).
- **v0.2**: NixOS overlay, GUI Slint, offline bundle (tải package
  trước, apply sau).
- **v0.3**: Resume-sau-interrupt cho các lần chạy dở (hôm nay thì
  rollback; sắp tới sẽ cho bạn tiếp tục từ nơi lần trước dừng).

Trước mắt, rất mong mọi người giúp test 0.1.0:

```bash
# Thử dry-run trước — nó thật sự không chạm gì.
vietime-installer install fcitx5-bamboo --dry-run

# Sẵn sàng thì:
vietime-installer install fcitx5-bamboo
```
