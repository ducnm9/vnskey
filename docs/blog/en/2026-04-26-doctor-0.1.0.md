# Introducing `vietime-doctor` 0.1.0 — diagnose Vietnamese typing on Linux

*Published 26 April 2026*

If you have ever typed Vietnamese on a Linux desktop, you know the
drill. You install an engine, you reboot, you open the app that
matters, and nothing happens. No tone marks. No composition preedit.
Just plain ASCII where there should have been "nước".

The failure is rarely mysterious, but it is always tedious to chase.
Is IBus running? Is Fcitx5 running? *Both?* Are `GTK_IM_MODULE`,
`QT_IM_MODULE`, `XMODIFIERS` all pointing at the same framework? Is
your Electron app launching with `--ozone-platform=wayland`, or is it
quietly falling back to XWayland and losing the IM context along the
way? Is your SDL game reading `SDL_IM_MODULE`?

Each of these has a one-line fix. Finding out *which* one-line fix is
the hard part.

`vietime-doctor` is a small CLI that does the finding for you.

## What it does

`vietime-doctor` is **diagnostic-only**. It inspects your system,
reports what it finds, and prints shell-ready remediation commands.
It never modifies state. You stay in control.

```sh
$ vietime-doctor report --plain
VietIME Doctor 0.1.0
Generated 2026-04-26T10:23:11Z

System
  distro:  Ubuntu 24.04
  session: wayland
  locale:  en_US.UTF-8
  desktop: GNOME

Input method frameworks
  ibus:   running (pid 1432), 2 engines
  fcitx5: running (pid 1788), 1 engine    ← conflict
  active: CONFLICT

Issues
  [Error]    VD002  Two IM frameworks are running simultaneously
             → VR002  Stop one of the conflicting daemons

Recommendations
  VR002  Stop one of the conflicting daemons
         $ systemctl --user stop ibus-daemon
         [review before running]

Exit: 2 (Error)
```

Three subcommands:

- `vietime-doctor report` — full Markdown / plain-text / JSON output.
- `vietime-doctor check`  — 1-line status and exit code 0/1/2.
- `vietime-doctor list`   — enumerates every detector and checker in
  the binary.

The 0/1/2 exit codes map onto severity: 0 = clean, 1 = at least one
Warn, 2 = at least one Error or Critical. Wire the `check` subcommand
into your dotfiles and you get a one-line Vietnamese-typing health
check every shell start.

## What it *doesn't* do

Doctor does not fix anything yet. It reports and recommends.

That is deliberate. The first release of any diagnostic tool should
earn the user's trust by being entirely read-only. Once the reports
are accurate on a wide range of distros — and after we ship the
Installer in 0.2 — Doctor's recommendations become the Installer's
playbook.

Explicitly out of scope for 0.1:

- No automatic fixes. `ibus restart` must still come from you.
- No desktop environment switching. If GNOME Wayland is dropping
  your IBus preedit, Doctor will tell you that and link to the
  Fcitx5 migration guide, but we will not flip your session under
  you.
- No cloud telemetry. Everything runs locally, PII is redacted by
  default, and the output is safe to paste into a GitHub issue.

## What it catches

15 classes of issue, each with a stable VD### identifier so output
stays citable across releases:

| ID     | Severity | Meaning                                                          |
| ------ | -------- | ---------------------------------------------------------------- |
| VD001  | Critical | No IM framework is active, but a Vietnamese engine is installed  |
| VD002  | Error    | Two IM frameworks running at once                                |
| VD003  | Error    | `GTK_IM_MODULE` / `QT_IM_MODULE` / `XMODIFIERS` disagree         |
| VD004  | Warn     | `SDL_IM_MODULE` unset (SDL apps won't compose Vietnamese)        |
| VD005  | Warn     | Engine package installed but not registered with the framework   |
| VD006  | Warn     | Wayland session with IBus (known to drop preedit)                |
| VD007  | Error    | Electron app launched without Ozone/Wayland                      |
| VD008  | Warn     | Chrome on Wayland without the Ozone flag                         |
| VD009  | Warn     | Conflicting env var definitions across `/etc/profile.d` files    |
| VD010  | Warn     | VS Code installed via Snap (no IM portal support)                |
| VD011  | Warn     | Flatpak app missing the IM portal declaration                    |
| VD012  | Info     | Legacy `INPUT_METHOD` variable empty                             |
| VD013  | Warn     | Fcitx5 addon needed for Vietnamese input is disabled             |
| VD014  | Warn     | UTF-8 locale missing or misconfigured                            |
| VD015  | Info     | No Vietnamese engine installed anywhere                          |

The `VD### → VR###` mapping in the report is the heart of the tool:
every diagnosis has a named, versioned remediation, and the commands
are safe to copy-paste.

Full reference lives in `docs/user/en/CHECKS.md`.

## Under the hood

Doctor is a ~4.5 MiB Rust binary with no runtime deps beyond libc.
It builds on stable and ships in three forms:

- AUR (`vietime-doctor-bin` and `vietime-doctor-git`)
- `.deb` (via `cargo-deb`) for Debian / Ubuntu
- Flatpak (`io.vietime.Doctor`) for Silverblue / Kinoite

16 detectors collect facts (distro, session, env vars from 5 sources,
IBus + Fcitx5 daemon state, installed engine packages, Electron /
Chromium command lines). 15 checkers then run a pure `(&Facts) →
Vec<Issue>` pass over the result.

Everything is covered by insta snapshot fixtures for Ubuntu 24.04,
Fedora 40, Arch, Debian 12 and openSUSE Tumbleweed, plus a
cargo-fuzz harness for the `/etc/environment` and `/etc/os-release`
parsers. The release binary budget is 8 MiB, enforced in CI.

## Next

**Phase 2 (0.2) brings the Installer.** Same design — a
recommendation-driven CLI — but now with `apply` verbs that execute
the VR### commands under your supervision. Think `vietime-doctor
fix VD002` with a dry-run preview and an atomic rollback.

**Phase 3 opens the GUI.** A small GTK frontend wrapping the same
detection core, for users who would rather click than paste commands
into a terminal.

In the meantime, you can install 0.1.0 today:

```sh
# Arch
yay -S vietime-doctor-bin

# Debian / Ubuntu
sudo dpkg -i vietime-doctor_0.1.0_amd64.deb

# Flatpak (coming to Flathub; local build today)
flatpak-builder --user --install build-dir \
    packaging/flatpak/io.vietime.Doctor.yml
```

Bug reports and distro-specific snapshots always welcome. PII is
redacted by default, so `vietime-doctor report --json` output is safe
to attach.
