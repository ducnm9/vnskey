# Introducing `vietime-installer` 0.1.0 — one-click Vietnamese typing on Linux

*Published 26 April 2026*

Six months ago we shipped `vietime-doctor` — a read-only diagnostic that
tells you *why* Vietnamese typing is broken on your Linux desktop.
Doctor is great at pinpointing the failure. What it explicitly does
not do is fix it for you. That was by design: Phase 1 was about
earning trust by never touching your system.

Today we're shipping the companion tool. `vietime-installer` 0.1.0 is
the piece that takes the Doctor's shell-ready remediation block and
just **runs it**, atomically, with a proper rollback trail.

From fresh Ubuntu install to working Vietnamese typing in roughly two
minutes:

```bash
vietime-installer install fcitx5-bamboo
# … one sudo prompt …
# Log out, log back in.
```

That's it.

## Why a second tool?

The shortest setup recipe for Fcitx5 + Bamboo on Ubuntu today is:

1. Uninstall or disable IBus.
2. `sudo apt install fcitx5 fcitx5-bamboo`.
3. Edit `/etc/environment` (or `~/.profile`) with five new env vars.
4. `im-config -n fcitx5`.
5. `systemctl --user enable --now fcitx5`.
6. Log out and log back in.
7. Open `fcitx5-configtool` and add Bamboo.
8. Try typing and pray.

Every one of these steps has a unique failure mode. Skip step 3 and
GTK won't speak to the daemon. Skip step 4 and `im-config` picks a
stale framework. Skip step 5 and Fcitx5 doesn't auto-start. Skip step
6 and GTK is still reading the old env vars from the previous login.

Installer collapses all of that into one command, records every
mutation in a snapshot manifest, and lets you undo the whole thing
with `vietime-installer uninstall`.

## What 0.1.0 ships

- **Four MVP combos**: `fcitx5-bamboo`, `fcitx5-unikey`, `ibus-bamboo`,
  `ibus-unikey`.
- **Debian-family distros**: Ubuntu 22.04+, Debian 12+, Pop!_OS 22.04+.
  Fedora and Arch land in v0.2 (the architecture is already
  distro-pluggable — only `AptOps` ships today).
- **Dry-run by default-discoverable**: `--dry-run` prints the plan
  without mutating anything.
- **Atomic rollback**: every run writes a snapshot at
  `~/.config/vietime/snapshots/<ts>/` with a TOML manifest, SHA-256-
  verified backups of modified config files, and a list of packages we
  added (so rollback only uninstalls *those*, not ones you had
  beforehand).
- **SIGINT-safe**: Ctrl+C between steps leaves the manifest flagged
  `incomplete = true`; `rollback --force` cleans up.
- **Never caches passwords**: the installer piggybacks on `sudo`'s own
  credential cache. It runs `sudo -v` once at the top of an install
  and lets sudo prompt you on the TTY.
- **Integrates with Doctor**: `vietime-installer verify` shells out to
  `vietime-doctor check` for the post-install smoke test.

## The design you don't see

The guts of the installer are three layers:

1. **Planner** — pure function `(PreState, Goal) -> Plan`. Takes a
   snapshot of the host system (reusing the Doctor's detectors) and a
   user goal, returns a structured list of `Step` variants. Fully
   deterministic, roundtrips through TOML, and has golden snapshot
   tests.
2. **Executor** — takes a `Plan` and runs it. The critical invariant:
   *record before mutate*. Every step's rollback artifact is written
   to the snapshot manifest before the mutation itself runs. A SIGKILL
   between the two leaves a no-op rollback target, never an
   unrecorded change.
3. **`PackageOps` trait** — async abstraction over the distro package
   manager. `AptOps` is the only implementation today. `DnfOps` and
   `PacmanOps` are one trait-impl each away.

Snapshots are content-addressed by SHA-256 sidecar files, so restoring
a 10-day-old backup catches bit-rot before it corrupts your
`/etc/environment`. Env-file edits are bracketed by markers:

```
# >>> VietIME managed start >>>
GTK_IM_MODULE=fcitx
QT_IM_MODULE=fcitx
XMODIFIERS=@im=fcitx
# <<< VietIME managed end <<<
```

Uninstalling removes *only* what's inside those markers. Your own
edits above and below the block are preserved verbatim.

## What's next

Phase 2 was about proving the architecture on one distro family. Phase
3 spreads it sideways:

- **INS-50 / INS-51**: `DnfOps` (Fedora) and `PacmanOps` (Arch).
- **v0.2**: NixOS overlay, Slint GUI, offline bundles (pre-download
  packages, apply later).
- **v0.3**: Resume-after-interrupt for partial runs (today we rollback;
  soon we'll let you continue from where the last run died).

For now, we'd love your help testing 0.1.0:

```bash
# Try it in dry-run first — it literally touches nothing.
vietime-installer install fcitx5-bamboo --dry-run

# Then, when you're ready:
vietime-installer install fcitx5-bamboo
```

File issues at <https://github.com/vietime/vietime/issues>. The
